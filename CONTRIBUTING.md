# 参与贡献（Hack_Agent）

感谢你愿意改进本仓库。

## 原则

- 变更尽量**小步、可审**：与安全/渗透相关的默认行为改动请说明风险与回滚方式。
- 提交前本地至少：`bun run build`，涉及核心路径时补充 `bun test` 或针对性测试。

## 安全

请勿在 issue/PR 中粘贴真实密钥、目标 URL 或未脱敏漏洞数据。敏感问题请按 [SECURITY.md](SECURITY.md) 联系。

## 分支与上游

本 fork 可能与 [Gitlawb/openclaude](https://github.com/Gitlawb/openclaude) 长期分叉；若需同步上游大版本，建议单独分支处理冲突后再合并主开发分支。
